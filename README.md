# OOPZ+

OOPZ+ 是一个面向 Windows 的 OOPZ 附属增强工具，核心目标是让多账号使用更顺手：保存已登录过的账号数据，在主程序、托盘菜单和插件浮层里快速切换账号。

> OOPZ+ 不修改 OOPZ 程序文件，不注入 DLL，不逆向登录协议。它只在当前 Windows 用户环境中备份、恢复 OOPZ 的本地账号数据，并提供更方便的入口。

## 主要功能

- 账号快速切换：保存已登录过的账号，一键关闭并重启 OOPZ，切换到目标账号。
- 插件模式：只打开 OOPZ 时，也能自动显示 OOPZ+ 头像浮层。
- 浮层布局：支持横排、竖排切换，通过右上角移动锚调整相对位置并自动保存。
- 登录页浮层：在 OOPZ 登录界面展示本地账号头像，可快速切号或引导登录一次。
- 主界面浮层：登录后在 OOPZ 主界面展示同样的账号头像入口。
- 托盘切号：右键托盘图标快速切换账号。
- 当前账号提示：当前登录账号在主程序、托盘和浮层中不可重复点击。
- OOPZ 路径识别：支持自动搜索 OOPZ 安装目录，也支持手动指定目录。
- 登录态包：可把单个或全部已保存登录态导出为统一的 `.oopz+` 文件，在其他客户端批量导入。
- 快捷分享：通过 Magic Wormhole 生成一次性代码，端到端加密发送全部已保存登录态，无需自建服务器，并可随时取消等待或发送。
- 安全保存：账号密码不写入明文配置文件，优先使用系统安全存储。
- 自动备份：切换前备份当前 OOPZ 数据，异常时可恢复最近一次备份。
- 自动更新：每半小时从 GitHub Release 检查新版，优先通过下载加速线路获取并校验安装包，失败时回退官方直链。

## 使用说明

### 1. 首次打开

打开 OOPZ+ 后，先在「概览」确认 OOPZ 路径是否已识别。未识别时可以点击自动搜索，或手动选择包含 `oopz.exe` 的目录。

### 2. 保存账号

在 OOPZ 中登录一次目标账号，然后回到 OOPZ+ 的「账号切换」页面刷新账号。保存成功后，该账号会显示为可快速切换。

### 3. 快速切号

在「账号切换」页面点击账号后面的快速切号按钮，或使用托盘菜单、插件浮层头像进行切换。

切换时 OOPZ+ 会：

1. 关闭当前 OOPZ。
2. 备份当前 OOPZ 本地数据。
3. 恢复目标账号的数据。
4. 重新启动 OOPZ。

### 4. 插件模式

在「插件模式」页面开启插件模式后，OOPZ+ 会安装一个轻量守护进程。之后即使没有手动打开 OOPZ+ 主窗口，只打开 OOPZ 也会自动出现账号头像浮层。

插件模式只显示贴附在 OOPZ 上的浮层，不显示 OOPZ+ 主窗口和托盘图标。只有用户手动打开 OOPZ+ 时，主窗口和托盘才会出现。

### 5. 导出和导入登录态

在「账号切换」页面点击「导出全部」，可把本机所有可切换账号打包为按时间命名的 `.oopz+` 文件。账号详情里的「导出」使用相同格式，但文件内只包含当前账号。

其他用户通过「导入」选择该文件后，OOPZ+ 会合并同 UID 账号并导入其余账号。旧版 `.json` 和 `.txt` 单账号导出文件仍可导入。

`.oopz+` 文件包含可用于登录的敏感本地状态，未进行密码加密。它不会包含 OOPZ+ 安全存储中的账号密码，但仍应只通过可信渠道发送，并在使用后妥善删除。

「快捷分享」会先生成全部账号登录态包，再通过 Magic Wormhole 公共服务生成 4 个词的一次性代码。接收方在「快捷导入」中输入该代码即可建立端到端加密连接；服务只负责双方发现和必要的流量中继，不能解密文件内容。分享方可在等待或发送过程中取消，10 分钟未完成也会自动失效。

## 状态标识

- 绿色对钩：该账号可快速切换。
- 黄色感叹号：该账号需要先登录一次。
- 当前登录：当前正在使用的账号，不会重复切换。

## 安全边界

OOPZ+ 的 v1.1.4 设计边界如下：

- 不修改 OOPZ 程序文件。
- 不注入 OOPZ 进程。
- 不调用或逆向 OOPZ 登录接口。
- 不在 JSON 配置中保存明文密码。
- 默认只支持当前 Windows 用户环境。
- 账号快速切换依赖已登录过的本地数据。

环境要求：

- Windows 10/11
- Node.js
- pnpm

pnpm install

## 常见问题

### 为什么某个账号显示需要登录一次？

说明 OOPZ+ 还没有保存到这个账号可用于快速切换的数据。请先在 OOPZ 中正常登录一次，再回到 OOPZ+ 刷新。

### 插件模式开启后为什么没有浮层？

先确认 OOPZ 正在运行，并在 OOPZ+ 的「插件模式」页面点击「修复环境」。如果刚安装或刚升级，建议重新打开 OOPZ。

### 切号会删除 OOPZ 原有数据吗？

默认不会。OOPZ+ 会在切换前备份当前数据，删除账号时也只删除 OOPZ+ 自己保存的数据，除非用户明确选择清理。

### 可以跨电脑使用吗？

可以使用账号导出导入功能迁移已保存的账号登录数据。导出的文件包含账号本地登录数据，请妥善保管。
